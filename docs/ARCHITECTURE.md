# معماری پروژه

## اصول

- ساختار پروژه بر اساس قابلیت‌ها (`feature-based`) است.
- هر قابلیت، صفحه‌ها و اجزای نمایشی خود را نگهداری می‌کند.
- سرویس‌های سراسری و مدل‌های مشترک در `core` قرار می‌گیرند.
- اجزای قابل استفاده مجدد و بدون منطق دامنه در `shared` قرار خواهند گرفت.
- صفحه‌ها با مسیریابی lazy بارگذاری می‌شوند.
- TypeScript و Angular template checking در حالت strict هستند.
- پردازش FAQ داخل مرورگر انجام می‌شود و وابستگی اجرایی به سرور ندارد.

## مرز پوشه‌ها

```text
src/app/
├── core/
│   ├── models/
│   └── services/
├── features/
│   └── assistant/
│       └── pages/
├── app.config.ts
├── app.routes.ts
└── app.component.ts
```

`core` نباید به `features` وابسته شود. قابلیت‌ها می‌توانند از `core` و `shared` استفاده کنند، اما نباید مستقیم به جزئیات داخلی قابلیت دیگری وابسته باشند.

## بک‌اند

API محلی در پوشه `server` قرار دارد و شامل ماژول‌های `auth`، `faqs` و `conversations` است. اطلاعات نسخه مستقل در فایل JSON محلی نگهداری می‌شود و دسترسی مسیرهای مدیریتی با JWT و نقش `admin` کنترل می‌شود.
