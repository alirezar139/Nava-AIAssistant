# نمایه مستندات

این پوشه مستندات محصول، معماری، API، اجرا، توسعه، امنیت و انتشار پروژه راهیار
را نگهداری می‌کند. ساختار مستندات بر اساس الگوی رایج پروژه‌های
نرم‌افزاری حرفه‌ای تنظیم شده است، اما متن‌ها برای خوانایی تیم فارسی‌زبان، فارسی
نوشته شده‌اند.

## استاندارد کیفیت مستندات

- هدف و مخاطب سند مشخص باشد.
- متن فارسی، روشن و قابل فهم نوشته شود.
- فرمان‌ها قابل copy/paste باشند.
- قراردادها، تنظیمات و وضعیت‌ها در جدول بیایند.
- جزئیات پیاده‌سازی فقط وقتی نوشته شود که برای نگهداری یا اجرا لازم است.
- مسیر فایل مالک هر رفتار ذکر شود.
- فرضیات تولید و استقرار بدون سند باقی نمانند.
- تغییر رفتار، API یا عملیات باید همراه با به‌روزرسانی مستندات باشد.

## چک‌لیست بازبینی مستندات

- لینک‌های README و docs معتبر هستند.
- نصب و اجرای پروژه از روی مستندات قابل انجام است.
- قراردادهای API با routeهای Express هم‌خوان هستند.
- وضعیت نیازمندی‌ها با کد فعلی مغایرت ندارد.
- فرمان‌های عملیاتی از نام واقعی scriptها استفاده می‌کنند.
- هیچ secret یا credential واقعی در مستندات نیست.
- مثال‌های commit با استاندارد پروژه سازگارند.
