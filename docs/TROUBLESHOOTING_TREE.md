# مستند درختواره عیب‌یابی دستیار

## هدف

درختواره عیب‌یابی مسیر مرحله‌ای گفت‌وگو را مشخص می‌کند. کاربر ابتدا مشکل را آزاد
توضیح می‌دهد، سپس دستیار حوزه و جزئیات را از طریق گزینه‌های درختواره مشخص می‌کند.
اگر پاسخ قابل اتکا در FAQ پیدا نشود، جریان جمع‌آوری اطلاعات تیکت و ارسال به سهند
شروع می‌شود.

## محل پیاده‌سازی

| بخش                     | مسیر                                                                          |
| ----------------------- | ----------------------------------------------------------------------------- |
| داده درختواره           | `src/assets/troubleshooting-tree.json`                                        |
| مدل TypeScript          | `src/app/core/models/troubleshooting-tree.models.ts`                          |
| سرویس بارگذاری و پیمایش | `src/app/core/services/troubleshooting-tree.service.ts`                       |
| مصرف در صفحه کاربر      | `src/app/features/assistant/pages/assistant-page/assistant-page.component.ts` |

## ساختار فایل درختواره

```ts
interface TroubleshootingTree {
  startNodeId: string;
  introNodeIds: string[];
  nodes: TroubleshootingTreeNode[];
  edges: TroubleshootingTreeEdge[];
}
```

- `startNodeId`: گره شروع مسیر تصمیم.
- `introNodeIds`: گره‌های معرفی/متن کمکی که در فایل مرجع وجود دارند.
- `nodes`: گره‌های قابل نمایش یا گره‌های واسط.
- `edges`: ارتباط بین گره‌ها؛ در صورت داشتن `label` همان متن به‌عنوان گزینه نمایش داده می‌شود.

نسخه فعلی فایل شامل ۳۰۶ گره و ۳۴۳ ارتباط است.

## جریان کاربر

1. دستیار در ابتدا از کاربر می‌خواهد مشکل را با چند جمله توضیح دهد.
2. متن اولیه کاربر در `treeTrail` ذخیره می‌شود.
3. دستیار از `startNodeId` وارد درختواره می‌شود و گزینه‌های گره را نمایش می‌دهد.
4. با انتخاب هر گزینه، مسیر طی‌شده در `treeTrail` ثبت می‌شود.
5. اگر گره نهایی به ثبت تیکت اشاره کند یا پاسخ قابل اتکا در FAQ پیدا نشود، جریان تیکت شروع می‌شود.
6. قبل از ارسال تیکت، اطلاعات زیر از کاربر گرفته می‌شود:
   - عنوان مشکل
   - سامانه یا ابزار
   - سناریو/فرآیند/جریان داده یا محیط سیتریکس
   - سریال یا شناسه
   - متن خطا
   - مستندات یا فایل Word
7. بعد از ثبت، دستیار رسید ثبت را به کاربر نمایش می‌دهد:
   - شماره تیکت داخلی با قالب `NAVA-00001`
   - شماره پیگیری داخلی با قالب `TRK-00001`
   - شماره تیکت سهند، اگر API سهند مقدار آن را برگرداند
   - شماره پیگیری سهند، اگر API سهند `trackingCode`، `trackingId`، `code` یا `requestId` برگرداند

## منطق پیمایش

`TroubleshootingTreeService` هنگام نمایش گره، گره‌های واسط کوتاه و بدون برچسب را
رد می‌کند تا کاربر با مرحله‌های اضافی و بدون معنا روبه‌رو نشود. این رفتار در
متد `resolveDisplayNode` پیاده‌سازی شده است.

```text
کاربر مشکل را توضیح می‌دهد
        ↓
نمایش حوزه‌های اصلی از startNodeId
        ↓
انتخاب گزینه‌ها و تکمیل treeTrail
        ↓
جستجو در FAQ با مسیر و متن مشکل
        ↓
پاسخ FAQ پیدا شد؟ ── بله → نمایش پاسخ
        ↓ خیر
جمع‌آوری اطلاعات تکمیلی
        ↓
نمایش فرم نهایی تیکت
        ↓
ارسال به API سهند / ثبت پرونده داخلی
        ↓
نمایش شماره تیکت و شماره پیگیری به کاربر
```

## نکته مخصوص سیتریکس

سیتریکس سناریو یا جریان داده ندارد و به‌عنوان محیط کاری شناخته می‌شود. در مسیر
سیتریکس، فیلد فرآیند به انتخاب محیط کاری تغییر می‌کند و کاربر باید یکی از این
موارد را مشخص کند:

- `star-da1`
- `star-da2`
- `star-da3`
- `star-da4`

## وضعیت فعلی و کارهای بعدی

- مصرف درختواره در صفحه کاربر پیاده‌سازی شده است.
- فایل درختواره فعلاً static asset است.
- ویرایشگر مدیریتی درختواره هنوز لازم است.
- اعتبارسنجی چرخه، گره بدون مقصد و گزینه تکراری باید قبل از ورود مدیریتی اضافه شود.
- در فاز بعد، فایل ثابت باید به API و مدل ذخیره‌سازی قابل مدیریت منتقل شود.
