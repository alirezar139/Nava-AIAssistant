# نمای کلی محصول

## خلاصه

راهیار یک سامانه داخلی پشتیبانی برای کاربران فارسی است. سامانه کاربر
را از درختواره عیب‌یابی عبور می‌دهد، پاسخ‌های تاییدشده FAQ را جست‌وجو می‌کند و
اگر مسئله حل نشود، تیکت سهند/Jira یا رسید داخلی ایجاد می‌کند.

## اهداف

- کاهش پرسش‌های تکراری با هدایت مرحله‌ای و FAQ تاییدشده.
- جمع‌آوری اطلاعات تشخیصی یکسان قبل از ارجاع به پشتیبانی.
- نمایش مرحله فعلی پیگیری برای تیم پشتیبانی.
- امکان اجرا در محیط داخلی و آفلاین.
- آماده‌سازی پایه قابل نگهداری برای SSO سازمانی و اتصال نهایی به سهند/Jira.

## خارج از دامنه نسخه فعلی

- چت‌بات عمومی یا هوش مصنوعی آزاد.
- جایگزینی کامل سامانه سهند/Jira.
- ویرایشگر گرافیکی مدیریتی برای درختواره.
- اتصال SSO سازمانی.
- دیتابیس تولیدی با migration و replication.

## نقش‌ها

| نقش         | نیاز اصلی                                                                   |
| ----------- | --------------------------------------------------------------------------- |
| کاربر نهایی | انتخاب حوزه مشکل، دریافت راهنمایی تاییدشده و گرفتن کد پیگیری در صورت ارجاع. |
| پشتیبان     | مشاهده مرحله پرونده و دریافت جزئیات ساختاریافته برای پیگیری.                |
| مدیر        | مدیریت FAQ، سرویس‌ها، گزارش‌ها و ارزیابی عملکرد پشتیبانی.                   |
| توسعه‌دهنده | نگهداری Angular، API و مرزهای integration.                                  |

## جریان اصلی کاربر

1. کاربر وارد سامانه می‌شود.
2. دستیار گزینه‌های اولیه درختواره را نمایش می‌دهد.
3. کاربر مسیر مشکل را انتخاب می‌کند.
4. سامانه FAQهای تاییدشده را جست‌وجو می‌کند.
5. اگر پاسخ قابل اتکا وجود داشته باشد، پاسخ نمایش داده و گفتگو ثبت می‌شود.
6. اگر پاسخ قابل اتکا وجود نداشته باشد، پرونده تشخیصی و تیکت ساخته می‌شود.
7. کاربر شماره پیگیری داخلی و در صورت وجود، اطلاعات سامانه مقصد را می‌بیند.

## قابلیت‌های فعلی

- احراز هویت محلی با JWT و CAPTCHA.
- نقش‌های مدیر و کاربر.
- مدیریت FAQ و ورود Excel.
- تعریف سرویس‌های مستقل توسط مدیر و نمایش سرویس‌های فعال برای کاربر.
- درختواره عیب‌یابی مرحله‌ای.
- ثبت گزارش گفتگو.
- ایجاد و تحلیل اولیه پرونده تشخیصی.
- ارسال اختیاری تیکت به سهند/Jira.
- داشبورد ارزیابی عملکرد شامل KPI، قیف پشتیبانی، روند ۷ روزه و هشدار مدیریتی.
- تنظیم تم برای هر کاربر، شامل Paint mode و عکس پروفایل.
- رابط فارسی برای صفحه کاربر، ادمین و ورود.

## فرضیات عملیاتی

- در توسعه، فرانت‌اند و بک‌اند در دو ترمینال جدا اجرا می‌شوند.
- LowDB برای ذخیره‌سازی محلی و داخلی استفاده می‌شود.
- در محیط توسعه ممکن است دسترسی سهند/Jira وجود نداشته باشد.
- حساب‌های پیش‌فرض فقط برای توسعه هستند.

## معیار موفقیت

- کاربر بتواند بدون ابهام مسیر پشتیبانی را طی کند.
- مسیر حل‌نشده به تیکت و رسید پیگیری برسد.
- مدیر بتواند FAQ و گزارش‌ها را مدیریت کند.
- مدیر بتواند سرویس عملیاتی جدید تعریف کند و کاربر همان سرویس را در بخش خودش اجرا کند.
- مدیر بتواند عملکرد پشتیبانی را با شاخص‌های قابل فهم ارزیابی کند.
- توسعه‌دهنده بتواند پروژه را از روی مستندات نصب، اجرا و build کند.
- رابط در تم روشن، تاریک، پالت‌ها و Paint mode خوانا بماند.
