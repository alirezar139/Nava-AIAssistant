# چک‌لیست انتشار

این چک‌لیست قبل از تحویل به QA، دمو، پشتیبانی یا محیط عملیاتی استفاده می‌شود.

## کنترل Git

- working tree تمیز باشد.
- branch با شاخه مقصد sync شده باشد.
- پیام‌های commit مطابق `docs/GIT_COMMIT_CONVENTION.md` باشند.
- تغییرات رفتاری همراه با مستندات commit شده باشند.

## Build و قالب‌بندی

اجرا کنید:

```powershell
npm run format:check
npm run build:all
```

خروجی مورد انتظار:

- Prettier خطای قالب‌بندی ندهد.
- build تولیدی Angular موفق شود.
- کامپایل TypeScript بک‌اند موفق شود.
- خروجی‌های `dist/nava-ai-assistant` و `server/dist` ساخته شوند.

## Smoke Test اجرا

API:

```powershell
npm run start:api
```

فرانت‌اند:

```powershell
npm start
```

بررسی کنید:

- `http://127.0.0.1:3000/api/health` مقدار `status: ok` بدهد.
- `http://localhost:4200/` باز شود.
- ورود مدیر با حساب توسعه کار کند.
- ورود کاربر با حساب توسعه کار کند.

Smoke test نسخه build شده:

```powershell
npm run start:webapp
```

بررسی کنید:

- `http://127.0.0.1:3000/` باز شود.
- refresh روی routeهای داخل برنامه خطای `404` ندهد.
- `http://127.0.0.1:3000/api/health` همچنان مقدار `status: ok` بدهد.
- `http://127.0.0.1:3000/manifest.webmanifest` قابل دریافت باشد.
- `http://127.0.0.1:3000/nava-service-worker.js` قابل دریافت باشد.

## Smoke Test کارکردی

- کاربر بتواند مسیر عیب‌یابی را انتخاب کند.
- کاربر بتواند به مرحله قبل برگردد.
- کاربر بتواند گفتگو را از ابتدا شروع کند.
- در صورت وجود پاسخ قابل اتکا، پاسخ FAQ نمایش داده شود.
- در صورت نبود پاسخ قابل اتکا، پرونده تشخیصی و رسید تیکت ساخته شود.
- رسید تیکت بعد از ثبت قابل مشاهده بماند.
- مدیر بتواند FAQها را ببیند و مدیریت کند.
- مدیر بتواند گزارش گفتگوها و پرونده‌ها را ببیند.
- پنل تم بتواند حالت روشن/تاریک، پالت، Paint color، کنتراست، حرکت و عکس پروفایل
  را تغییر دهد.

## Smoke Test ظاهری

- چیدمان فارسی درست و خوانا باشد.
- در صفحه کاربر متن‌ها روی desktop و mobile روی هم نیفتند.
- متن هدر داشبورد ادمین روی پس‌زمینه تیره خوانا بماند.
- کارت‌های آمار داشبورد مرتب و هم‌تراز باشند.
- صفحه ورود از tokenهای سراسری تم استفاده کند.
- عکس پروفایل درست crop شود و اندازه header یا پیام را تغییر ندهد.
- گزینه نصب PWA در Chrome یا Edge نمایش داده شود.
- آیکن نصب‌شده در Start Menu و Taskbar خوانا باشد.

## امنیت و تنظیمات

- حساب‌های پیش‌فرض خارج از توسعه استفاده نشوند.
- `JWT_SECRET` برای محیط مشترک تنظیم شده باشد.
- اطلاعات سهند/Jira فقط از environment variable خوانده شوند.
- secret در repository commit نشده باشد.
- سیاست backup داده‌های LowDB قبل از استفاده داخلی واقعی مشخص باشد.
