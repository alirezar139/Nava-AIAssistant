# نیازمندی‌ها

## هدف

سامانه راهیار باید کاربر فارسی را در یک جریان پشتیبانی مرحله‌ای هدایت
کند، FAQهای تاییدشده را بررسی کند و در صورت حل نشدن مسئله، پرونده تشخیصی و تیکت
پشتیبانی ایجاد کند.

## دامنه

داخل دامنه نسخه فعلی:

- وب‌اپ داخلی و محلی.
- تجربه کاربری فارسی برای کاربر و مدیر.
- درختواره عیب‌یابی مرحله‌ای.
- جست‌وجو و مدیریت FAQ.
- گزارش گفتگوها.
- ایجاد پرونده تشخیصی.
- ارسال اختیاری تیکت به سهند/Jira.
- تعریف سرویس‌های عملیاتی توسط مدیر و نمایش سرویس‌های فعال برای کاربر.
- ارزیابی عملکرد پشتیبانی در پنل مدیریت.
- تنظیم تم و عکس پروفایل برای هر کاربر.

خارج از دامنه نسخه فعلی:

- SSO سازمانی.
- ویرایشگر گرافیکی درختواره در پنل مدیر.
- migration تولیدی دیتابیس.
- چت‌بات عمومی یا پاسخ آزاد هوش مصنوعی.

## نیازمندی‌های کارکردی

### FR-01 احراز هویت

سامانه باید کاربر را با نام کاربری، رمز عبور و CAPTCHA احراز هویت کند. نسخه فعلی
از کاربر محلی و JWT استفاده می‌کند. اتصال SSO سازمانی برای فاز بعدی باقی می‌ماند.

### FR-02 کنترل دسترسی مبتنی بر نقش

سامانه باید نقش مدیر و کاربر را جدا کند. عملیات مدیریتی باید در API محافظت شوند.

### FR-03 مدیریت FAQ

مدیر باید بتواند FAQ ایجاد، ویرایش، حذف، حذف گروهی و import کند. داده Excel باید
به question، answer، category و keywords تبدیل شود.

### FR-04 عیب‌یابی مرحله‌ای

کاربر باید از گره شروع درختواره هدایت شود. کاربر باید بتواند به مرحله قبل برگردد
و گفتگو را از ابتدا شروع کند.

### FR-05 بررسی FAQ

دستیار باید با استفاده از مسیر انتخاب‌شده و زمینه کاربر، FAQهای تاییدشده را
جست‌وجو کند. پاسخ قابل اتکا باید قبل از ثبت تیکت نمایش داده شود.

### FR-06 ثبت خودکار تیکت

اگر FAQ پاسخ قابل اتکا ندهد، سامانه باید از مسیر انتخاب‌شده payload تشخیصی بسازد
و پرونده داخلی ایجاد کند.

### FR-07 اتصال سهند/Jira

اگر تنظیمات سامانه مقصد وجود داشته باشد، بک‌اند باید پرونده را به endpoint
سهند/Jira ارسال کند. اگر اتصال تنظیم نشده باشد، رسید داخلی باید همچنان ساخته شود.
مدیر باید بتواند آدرس سرویس، هدر `Authorization`، هدر اضافی `Auth`، شناسه
پروژه/Service Desk، شناسه RequestType و نگاشت Nodeهای درختواره به RequestType را
از پنل پیکربندی تغییر دهد. سامانه باید هنگام ثبت خودکار تیکت، نوع درخواست را از
نگاشت Node انتخاب‌شده استخراج کند و اگر نگاشت موجود نبود از مقدار عمومی استفاده
کند.

### FR-08 گزارش مدیریتی

مدیر باید بتواند گفتگوها و پرونده‌های تشخیصی را همراه با اطلاعات کاربر ببیند.

### FR-09 شخصی‌سازی

کاربر باید بتواند mode تم، پالت، رنگ Paint، کنتراست، motion و عکس پروفایل خود را
تنظیم کند. تنظیمات باید مخصوص همان کاربر ذخیره شوند.

### FR-10 نصب رابط به‌صورت PWA

کاربر باید بتواند رابط راهیار را در مرورگرهای پشتیبانی‌شده مثل Chrome و Microsoft
Edge به‌صورت PWA نصب کند. نصب PWA جایگزین اجرای بک‌اند نیست و سرور Node باید
همچنان در دسترس باشد.

### FR-11 کاتالوگ سرویس‌های سامانه

مدیر باید بتواند سرویس عملیاتی جدید تعریف کند، هدف آن را بنویسد، آدرس، متد،
هدرها و قالب بدنه را تنظیم کند و مشخص کند آیا سرویس برای کاربر نمایش داده شود
یا خیر. سرویس‌های فعال باید در صفحه کاربر بخش خودشان را بسازند و اجرای آن‌ها از
بک‌اند انجام شود تا اطلاعات محرمانه سرویس در مرورگر نمایش داده نشود.

### FR-12 داشبورد ارزیابی عملکرد

مدیر باید بتواند در یک تب مستقل وضعیت عملکرد پشتیبانی را ببیند. این داشبورد باید
شاخص‌های کلیدی گفتگو، پوشش FAQ، پرونده‌های پشتیبانی، ارسال سهند، سرویس‌های فعال،
روند ۷ روزه، قیف پشتیبانی، سهم دسته‌های FAQ و هشدارهای مدیریتی را نمایش دهد.

## معیارهای پذیرش

- کاربر بتواند وارد شود و جریان دستیار را شروع کند.
- کاربر بتواند گزینه‌های درختواره را بدون نیاز به تایپ آزاد انتخاب کند.
- کاربر بتواند یک مرحله به عقب برگردد و گفتگو را restart کند.
- اگر FAQ پاسخ قابل اتکا داشت، پاسخ تاییدشده نمایش داده شود.
- اگر مسیر حل نشد، پرونده تشخیصی و رسید تیکت ایجاد شود.
- مدیر بتواند FAQها را مدیریت کند.
- مدیر بتواند سرویس جدید تعریف، تست، ویرایش، حذف و برای کاربر فعال کند.
- مدیر بتواند تب ارزیابی عملکرد را باز کند و شاخص‌های پشتیبانی را بدون پردازش دستی ببیند.
- کاربر بتواند سرویس فعال‌شده توسط مدیر را از بخش خودش اجرا کند.
- مدیر بتواند گزارش گفتگوها و پرونده‌ها را ببیند.
- تغییر تم روی صفحه کاربر، ادمین، ورود و اجزای مشترک اعمال شود.
- `npm run format:check` پاس شود.
- `npm run build:all` پاس شود.

## تصمیم‌های باز

- سرویس نهایی احراز هویت سازمانی.
- قرارداد نهایی endpoint سهند/Jira و فیلدهای اجباری.
- دیتابیس تولیدی و سیاست backup.
- فرآیند مالکیت و تایید محتوای FAQ و درختواره.
- دامنه ویرایشگر مدیریتی درختواره در فاز بعد.
