# Nava AI Assistant

دستیار Nava یک وب‌اپ فارسی برای پشتیبانی کاربران، مدیریت FAQ، عیب‌یابی مرحله‌ای
و ثبت خودکار تیکت سهند/Jira است. کاربر مسیر مشکل را از درختواره انتخاب می‌کند،
سامانه پاسخ‌های تاییدشده FAQ را بررسی می‌کند و اگر پاسخ قابل اتکا پیدا نشود،
پرونده پشتیبانی و رسید پیگیری ساخته می‌شود.

## قابلیت‌ها

- رابط فارسی برای کاربر و مدیر.
- درختواره عیب‌یابی در `src/assets/troubleshooting-tree.json`.
- جست‌وجوی پاسخ در پایگاه دانش FAQ.
- ثبت خودکار تیکت سهند/Jira در صورت نبود پاسخ قابل اتکا.
- روندنمای وضعیت پیگیری برای پشتیبان.
- داشبورد مدیریت FAQ، ورود Excel، گزارش گفتگوها و پرونده‌ها.
- تنظیم تم برای هر کاربر شامل روشن/تاریک، پالت‌ها، Paint mode و عکس پروفایل.
- ذخیره‌سازی محلی با LowDB برای توسعه و اجرای داخلی.
- API با Express و کنترل دسترسی مبتنی بر JWT و نقش کاربر.

## شروع سریع

نصب وابستگی‌ها:

```powershell
npm install
npm --prefix server install
```

اجرای API در ترمینال اول:

```powershell
npm run start:api
```

اجرای رابط Angular در ترمینال دوم:

```powershell
npm start
```

آدرس برنامه:

```text
http://localhost:4200/
```

در حالت توسعه، درخواست‌های `/api` از طریق `proxy.conf.json` به API محلی منتقل
می‌شوند.

بررسی سلامت API:

```text
http://127.0.0.1:3000/api/health
```

## اجرای وب‌اپ روی ویندوز

برای اجرای نسخه build شده با یک ترمینال، خروجی فرانت‌اند و بک‌اند را بسازید و
سپس سرور تولیدی را اجرا کنید:

```powershell
npm run build:webapp
npm run start:webapp
```

در این حالت Express فایل‌های Angular را هم سرو می‌کند و برنامه از این آدرس باز
می‌شود:

```text
http://127.0.0.1:3000/
```

روی ویندوز می‌توانید اسکریپت آماده را اجرا کنید. اگر خروجی build وجود نداشته
باشد، اسکریپت ابتدا build را می‌سازد و بعد وب‌اپ را بالا می‌آورد:

```powershell
npm run webapp:windows
```

## حساب‌های توسعه

| نقش   | نام کاربری | رمز عبور    |
| ----- | ---------- | ----------- |
| مدیر  | `admin`    | `Admin@123` |
| کاربر | `user`     | `User@123`  |

این حساب‌ها فقط برای توسعه هستند. قبل از هر استقرار مشترک یا واقعی، رمزها و
secretها باید تغییر کنند.

## تنظیمات محیطی

API این متغیرها را می‌خواند:

| متغیر               | اجباری              | توضیح                               |
| ------------------- | ------------------- | ----------------------------------- |
| `PORT`              | خیر                 | پورت API. مقدار پیش‌فرض `3000` است. |
| `JWT_SECRET`        | برای محیط واقعی بله | کلید امضای JWT.                     |
| `SAHAND_TICKET_URL` | خیر                 | آدرس ثبت تیکت سهند/Jira.            |
| `SAHAND_API_KEY`    | خیر                 | توکن Bearer برای سامانه مقصد.       |

اگر `SAHAND_TICKET_URL` تنظیم نشده باشد، ثبت تیکت با رسید داخلی انجام می‌شود تا
جریان پشتیبانی در محیط آفلاین هم قابل تست باشد.

## کنترل کیفیت

قبل از تحویل تغییرات اجرا کنید:

```powershell
npm run format:check
npm run build:all
```

اگر build در ویندوز هنگام حذف فایل‌های `dist` با خطای `EPERM` متوقف شد، dev
server یا برنامه‌ای که فایل را نگه داشته ببندید و build را دوباره اجرا کنید.
